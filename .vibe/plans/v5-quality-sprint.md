```bash
vibe route --slash "/vibe-analyze --deep"
```

### Task 1.3: 统一 Hook 脚本生成（2天）

**现状**：三份 hook 脚本行为不一致——
- `claude_code.py` 走 Jinja2 模板（但模板可能过期）
- `opencode.py` 硬编码 113 行 bash
- `kimi_cli.py` 硬编码另一份 109 行 bash

**方案**：统一到一个 Jinja2 模板 `templates/shared/hooks/vibesop-route.sh.j2`，所有 Adapter 共用。

```jinja2
#!/bin/bash
# VibeSOP Route Hook for {{ platform }}
# Generated by VibeSOP v{{ version }}

set -e

INPUT=$(cat)
# Fallback: try jq first, then grep
QUERY=""
if command -v jq &> /dev/null; then
    QUERY=$(echo "$INPUT" | jq -r '.user_prompt // empty' 2>/dev/null || true)
fi
if [ -z "$QUERY" ]; then
    # Plain text input (no JSON wrapper)
    QUERY="$INPUT"
fi
# ... rest of shared logic ...
```

**验收标准**：
- [ ] 三个平台的 hook 脚本由同一模板生成
- [ ] Hook 脚本在有 `jq` 和无 `jq` 环境下均可工作
- [ ] 新增 `tests/adapters/test_hook_templates.py` 验证模板渲染

### Task 2.3: Badge 系统持久化（1天）

**现状**：`core/badges.py` 仅在 CLI 中 `print` 徽章提示，无持久化。

**方案**：徽章状态存储到 `.vibe/badges.json`。

```python
class BadgeManager:
    def award(self, badge_type: str, user_id: str = "default") -> bool:
        """Award a badge. Returns True if newly awarded."""
        badges = self._load()
        if badge_type in badges.get(user_id, {}):
            return False  # Already awarded
        badges.setdefault(user_id, {})[badge_type] = {
            "awarded_at": datetime.now().isoformat(),
            "reason": self.BADGE_REASONS.get(badge_type, "")
        }
        self._save(badges)
        return True
```

**验收标准**：
- [ ] 徽章持久化到文件
- [ ] 同一徽章不重复颁发
- [ ] `vibe badges list` 命令可查看

```python
# unified.py: _get_candidates()
def _get_candidates(self, ...):
    candidates = [...]
    # 新增过滤
    candidates = [c for c in candidates 
                  if c.lifecycle != SkillLifecycleState.ARCHIVED]
    
    for c in candidates:
        if c.lifecycle == SkillLifecycleState.DEPRECATED:
            c.add_warning(f"Skill {c.skill_id} is deprecated")
```

### Task 3.2: 多意图检测引入语义分析（3天）

**现状**：`orchestration/multi_intent_detector.py` 仅依赖正则匹配 "然后|接着|以及"。

**方案**：引入 LLM 辅助的多意图分解（降级到正则）。

```python
class TaskDecomposer:
    def decompose(self, query: str) -> list[SubTask]:
        # 1. 先用正则快速检查
        if not self._has_multi_intent_markers(query):
            return [SubTask(query=query)]  # 单意图
        
        # 2. 如果 LLM 可用，用 LLM 分解
        if self._llm and self._llm.is_available():
            return self._llm_decompose(query)
        
        # 3. 降级到正则分段
        return self._regex_segment(query)
```

**依赖 Task 2.1**：LLM 注入（`set_llm(agent_llm)`）必须可用。

**验收标准**：
- [ ] `TaskDecomposer.decompose("分析架构并优化性能")` 返回 2+ 子任务
- [ ] 无 LLM 时降级到正则分段
