## 📝 决策记录

### DEC-001: 采用工作流编排架构
**日期**: 2026-04-04
**决策**: 采用类似 Oh-My-Codex 的工作流编排架构
**原因**: 提升用户体验，支持复杂场景
**影响**: Phase 1-3 核心功能

### DEC-002: 保持 Pydantic v2 类型系统
**日期**: 2026-04-04
**决策**: 继续使用 Pydantic v2 作为核心数据模型
**原因**: 运行时验证 + 类型安全的优势
**影响**: 所有模块的数据模型

### DEC-003: 状态存储采用文件系统
**日期**: 2026-04-04
**决策**: 优先使用文件系统作为状态存储后端
**原因**: 简单、可靠、跨平台
**影响**: Phase 3 状态管理
