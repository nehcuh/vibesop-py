## Overview

The `vibesop.core.algorithms` module provides reusable, pure mathematical/detection algorithms that can be used by any skill. These are infrastructure components awaiting integration into skills.

## Available Algorithms

### 1. Ambiguity Detection (`interview/ambiguity`)

Computes ambiguity scores across multiple dimensions. Useful for:
- Interview preparation
- Requirement analysis
- Code review ambiguity detection

**Usage**:
```python
from vibesop.core.algorithms import compute_ambiguity, DimensionScore

result = compute_ambiguity(
    DimensionScore(name="clarity", score=0.8),
    DimensionScore(name="completeness", score=0.5),
    DimensionScore(name="specificity", score=0.6),
    DimensionScore(name="consistency", score=0.4),
    DimensionScore(name="testability", score=0.7),
)

print(f"Ambiguity: {result.ambiguity:.2f}")  # 0.0-1.0, higher = more ambiguous
print(f"Weakest: {result.weakest_dimension.name}")  # Dimension needing most attention
```

**Dimensions**:
- `clarity`: How clear is the statement?
- `completeness`: Is information missing?
- `specificity`: Are details concrete or vague?
- `consistency`: Does it contradict itself?
- `testability`: Can it be verified?

### 2. AI Slop Detection (`ralph/deslop`)

Detects common AI-generated code patterns ("slop") that may indicate:
- Over-engineering
- Generic solutions
- Missing context awareness

**Usage**:
```python
from vibesop.core.algorithms import scan_file

report = scan_file("path/to/code.py")

if report.has_slop:
    print(f"Found {len(report.patterns)} slop patterns:")
    for pattern in report.patterns:
        print(f"  - {pattern.name}: {pattern.description}")
        print(f"    Line {pattern.line_number}: {pattern.line snippet}")
```

**Detected Patterns**:
- Empty docstrings
- TODO comments without context
- Generic exception handling
- Unused imports
- Overly abstract class hierarchies

### Example 2: Interview Prep Skill

```python
from vibesop.core.algorithms import compute_ambiguity, DimensionScore

def assess_question(question: str) -> str:
    """Assess if an interview question is clear enough."""
    scores = analyze_question_dimensions(question)
    result = compute_ambiguity(*scores)

    if result.ambiguity > 0.5:
        return f"Question is ambiguous (weakest: {result.weakest_dimension.name})"
    return "Question is clear and specific"
```
