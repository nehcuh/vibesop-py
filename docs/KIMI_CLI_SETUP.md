# Kimi CLI Setup Guide

> **Last Updated**: 2026-04-19
> **Status**: ✅ Working

## Overview

VibeSOP provides a Kimi Code CLI adapter that generates complete configuration files including:
- `config.toml` - Main configuration with providers, models, and skills
- `skills/*/SKILL.md` - Individual skill definitions
- `README.md` - Usage guide and skill catalog

## Quick Start

### 1. Build Configuration

```bash
vibe build kimi-cli
```

This generates the complete configuration in `.vibe/dist/kimi-cli/`.

### 2. Deploy to Kimi CLI

```bash
# Deploy config
cp .vibe/dist/kimi-cli/config.toml ~/.kimi/config.toml

# Deploy skills
rm -rf ~/.kimi/skills
cp -r .vibe/dist/kimi-cli/skills ~/.kimi/
```

### 3. Configure API Key

Edit `~/.kimi/config.toml` and add your API key:

```toml
[providers.kimi-for-coding]
type = "kimi"
base_url = "https://api.kimi.com/coding/v1"
api_key = "sk-your-actual-api-key-here"  # Uncomment and set
```

### 4. Verify Installation

```bash
kimi --version
# Output: kimi, version 1.36.0

# List available skills
ls ~/.kimi/skills/
```

## Configuration Structure

### Providers Section

```toml
[providers.kimi-for-coding]
type = "kimi"
base_url = "https://api.kimi.com/coding/v1"
api_key = "sk-xxx"
```

### Models Section

```toml
[models.kimi-for-coding]
provider = "kimi-for-coding"  # References provider key above
model = "kimi-for-coding"
max_context_size = 262144
```

## Troubleshooting

### Error: "Provider not found"

**Cause**: Missing `[providers.*]` section or incorrect provider reference in models.

**Fix**: Ensure both sections exist and models correctly reference providers:

```toml
[providers.kimi-for-coding]
type = "kimi"
base_url = "..."

[models.kimi-for-coding]
provider = "kimi-for-coding"  # Must match provider key
```

### Error: "Control characters not allowed in strings"

**Cause**: Multi-line descriptions with newlines in TOML strings.

**Fix**: VibeSOP automatically escapes strings by:
- Removing newlines (replacing with spaces)
- Escaping quotes (`"` → `\"`)
- Collapsing multiple spaces

Generated by VibeSOP's `_escape_toml_string()` method.

### Skills Not Loading

**Check**:
1. Skills directory exists: `~/.kimi/skills/`
2. Each skill has `SKILL.md` file
3. `merge_all_available_skills = true` in config.toml

## Adapter Implementation

The Kimi CLI adapter is implemented in:
- `src/vibesop/adapters/kimi_cli.py`
- `src/vibesop/adapters/base.py` (base class)

Key methods:
- `render_config()` - Generates complete config with skills
- `_generate_config()` - Creates TOML content with providers, models, skills
- `_escape_toml_string()` - Safely escapes multi-line strings for TOML
