## AI-Powered Skill Orchestration

**MANDATORY: ALWAYS call `vibe route` before starting ANY non-trivial task**
This is NOT optional — routing ensures the correct skill(s) are loaded for the task.

### Workflow (Orchestration Mode — Default)

**Step 1**: Call routing to analyze intent and get execution plan
```bash
vibe route "<user_request>"
```

VibeSOP will:
1. Detect if your request contains multiple intents
2. Decompose complex requests into sub-tasks
3. Build an execution plan with optimal skill selection
4. Show a summary for confirmation

**Step 2**: Confirm or edit the execution plan
- The CLI will show detected steps and ask for confirmation
- You can edit, use single-skill fallback, or skip

**Step 3**: Execute skills in order
```markdown
read skills/<matched-skill>/SKILL.md
```

For orchestrated plans, execute each step sequentially or in parallel as indicated.

**Step 4**: Run verification after completion

### Single Intent Mode

For simple, single-intent requests, VibeSOP automatically falls back to single-skill routing:

```bash
vibe route "debug this error"
# → Single skill match: systematic-debugging (95%)
```

### Multi Intent Mode

For complex requests, VibeSOP automatically orchestrates:

```bash
vibe route "analyze architecture and write tests"
# → Orchestrated plan:
#   Step 1: riper-workflow — architecture analysis
#   Step 2: superpowers/test — test generation
```

### CLI Quick Commands

When the user types a `/vibe-*` command, execute it directly with `vibe route --slash`:
```bash
vibe route --slash "/vibe-help"
vibe route --slash "/vibe-list"
vibe route --slash "/vibe-analyze --deep"
vibe route --slash "/vibe-install gstack"
```

Available quick commands:
- `/vibe-help` — Show all available commands
- `/vibe-list` — List installed skills and available packs
- `/vibe-route <query>` — Force trigger routing with transparency
- `/vibe-install <pack>` — Install a skill pack
- `/vibe-analyze [--deep]` — Analyze project architecture
- `/vibe-evaluate [--skill <id>]` — Evaluate skill quality
- `/vibe-orchestrate <query>` — Multi-skill orchestration

### Skill Lifecycle Management

Skills have lifecycle states: `draft → active → deprecated → archived`

```bash
# List all skills with states
vibe skill list

# Enable/disable skills
vibe skill enable <skill-id>
vibe skill disable <skill-id>

# Check skill status
vibe skill status <skill-id>
```
